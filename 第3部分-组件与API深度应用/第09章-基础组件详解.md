## 第9章 基础组件详解

微信小程序提供了一套丰富的内置组件，类似于 HTML 中的标签体系，但命名和用法有显著差异。对于 Java Web 开发者来说，可以将其类比为 **Thymeleaf 模板标签** 或 **JSP 标签库** —— 它们都是框架封装好的 UI 原语，通过数据绑定驱动渲染。

### 9.1 视图容器组件

#### view —— 最基础的容器

`view` 类似 HTML 中的 `<div>`，是最常用的布局容器。它支持 flex 布局，是构建页面的骨架。

```xml
<!-- pages/index/index.wxml -->
<view class="container">
  <view class="header">新闻资讯</view>
  <view class="content">
    <view class="news-item" wx:for="{{newsList}}" wx:key="id">
      <text>{{item.title}}</text>
    </view>
  </view>
</view>
```

```css
/* pages/index/index.wxss */
.container {
  display: flex;
  flex-direction: column;
  height: 100vh;
}
.header {
  height: 90rpx;
  line-height: 90rpx;
  text-align: center;
  background-color: #07c160;
  color: #fff;
}
.content {
  flex: 1;
  padding: 20rpx;
}
```

> **类比理解**：`view` 就像 Spring MVC 中 Thymeleaf 的 `<div th:each="item : ${list}">`，负责承载循环渲染的内容块。区别在于小程序通过 `wx:for` 指令在 WXML 中直接声明循环，而非在服务端模板引擎中完成。

#### scroll-view —— 可滚动视图区域

`scroll-view` 用于实现区域内滚动，支持横向和纵向滚动，以及下拉刷新和触底加载。

```xml
<!-- 纵向滚动 + 下拉刷新 + 触底加载 -->
<scroll-view
  scroll-y
  style="height: 600rpx;"
  bindrefresherrefresh="onPullDownRefresh"
  refresher-enabled
  refresher-triggered="{{isRefreshing}}"
  bindscrolltolower="loadMore"
  lower-threshold="100"
>
  <view wx:for="{{articles}}" wx:key="id" class="article-item">
    <text>{{item.title}}</text>
  </view>
  <view wx:if="{{loading}}" class="loading-tip">加载中...</view>
</scroll-view>
```

```javascript
// pages/index/index.js
Page({
  data: {
    articles: [],
    page: 1,
    isRefreshing: false,
    loading: false,
    hasMore: true
  },

  onPullDownRefresh() {
    this.setData({ page: 1, hasMore: true })
    this.fetchArticles(true)
  },

  loadMore() {
    if (this.data.hasMore && !this.data.loading) {
      this.setData({ page: this.data.page + 1 })
      this.fetchArticles(false)
    }
  },

  async fetchArticles(isRefresh) {
    if (isRefresh) this.setData({ isRefreshing: true })
    else this.setData({ loading: true })

    const res = await wx.request({
      url: `https://api.example.com/articles?page=${this.data.page}&size=10`,
      method: 'GET'
    })

    const newArticles = res.data.content
    this.setData({
      articles: isRefresh ? newArticles : this.data.articles.concat(newArticles),
      hasMore: newArticles.length >= 10,
      isRefreshing: false,
      loading: false
    })
  }
})
```

> **Java开发者注意点**：`scroll-view` 的下拉刷新与页面的 `onPullDownRefresh` 生命周期是**两套独立机制**。前者作用于组件内部区域，后者作用于整个页面。在内容资讯类应用中，推荐使用 `scroll-view` 的 `refresher` 功能，因为它可以与页面其他固定布局（如底部 TabBar、顶部搜索栏）共存，而不会产生滚动冲突。

#### swiper —— 滑块视图容器（轮播图）

`swiper` 是内容资讯类小程序的核心组件之一，用于实现首页轮播图、图片画廊等场景。

```xml
<!-- pages/index/index.wxml -->
<swiper
  class="banner-swiper"
  indicator-dots="{{true}}"
  indicator-color="rgba(255,255,255,0.5)"
  indicator-active-color="#ffffff"
  autoplay="{{true}}"
  interval="3000"
  circular="{{true}}"
  duration="500"
  bindchange="onSwiperChange"
>
  <swiper-item wx:for="{{banners}}" wx:key="id">
    <image
      src="{{item.imageUrl}}"
      mode="aspectFill"
      class="banner-image"
      bindtap="onBannerTap"
      data-article-id="{{item.articleId}}"
    />
  </swiper-item>
</swiper>
```

```css
.banner-swiper {
  width: 100%;
  height: 360rpx;
}
.banner-image {
  width: 100%;
  height: 100%;
}
```

```javascript
// pages/index/index.js
Page({
  data: {
    banners: [
      { id: 1, imageUrl: '/images/banner1.jpg', articleId: '101' },
      { id: 2, imageUrl: '/images/banner2.jpg', articleId: '102' },
      { id: 3, imageUrl: '/images/banner3.jpg', articleId: '103' }
    ],
    currentSwiperIndex: 0
  },

  onSwiperChange(e) {
    this.setData({ currentSwiperIndex: e.detail.current })
  },

  onBannerTap(e) {
    const articleId = e.currentTarget.dataset.articleId
    wx.navigateTo({ url: `/pages/article/detail?id=${articleId}` })
  }
})
```

> **类比理解**：`swiper` 类似于前端常见的 Swiper.js 或 Bootstrap Carousel，但它是小程序原生组件，性能更优。对于 Java 开发者，可以把它想象为一个内置的"轮播图 JSP 标签"，只需传入数据列表即可渲染。

### 9.2 基础内容组件

#### text —— 文本组件

```xml
<!-- 基础文本 -->
<text>这是一段普通文本</text>

<!-- 可选中文本 -->
<text user-select>{{article.content}}</text>

<!-- 文本拼接 -->
<text>{{greeting}}，{{username}}！欢迎回来。</text>
```

> **注意**：`<text>` 组件内部只能包含文本，不能嵌套其他组件。如果需要富文本渲染，请使用 `rich-text`。

#### rich-text —— 富文本渲染

`rich-text` 是内容资讯类应用的关键组件，用于渲染后端返回的 HTML 格式文章内容。

```xml
<rich-text nodes="{{articleHtml}}" bindtap="onRichTextTap"></rich-text>
```

```javascript
Page({
  data: {
    articleHtml: ''
  },

  async loadArticle(id) {
    const res = await wx.request({
      url: `https://api.example.com/articles/${id}`,
      method: 'GET'
    })
    // 后端返回的 HTML 内容，通常由富文本编辑器（如 TinyMCE）生成
    this.setData({ articleHtml: res.data.content })
  },

  onRichTextTap(e) {
    // 可以捕获富文本内部的点击事件
    console.log('点击了富文本区域', e.detail)
  }
})
```

> **Java开发者注意点**：`rich-text` 使用的是**小程序自定义的节点树**，而非浏览器 DOM。它支持的 HTML 标签有限（如 `div`、`span`、`img`、`a`、`p`、`h1-h6` 等），不支持 `<script>`、`<style>`、`<iframe>` 等标签。如果你的后端使用 TinyMCE 或 CKEditor 生成 HTML，建议在 Java 端做一次过滤，移除不支持的标签和属性，避免渲染异常。
>
> 后端过滤示例（Java）：
> ```java
> public String sanitizeHtml(String rawHtml) {
>     Whitelist whitelist = Whitelist.relaxed()
>         .removeTags("script", "style", "iframe", "object", "embed");
>     return Jsoup.clean(rawHtml, whitelist);
> }
> ```

### 9.3 表单组件

#### button —— 按钮

```xml
<!-- 普通按钮 -->
<button type="primary" bindtap="onSubmit">提交</button>

<!-- 开放能力按钮：获取手机号 -->
<button
  open-type="getPhoneNumber"
  bindgetphonenumber="onGetPhoneNumber"
>
  获取手机号
</button>

<!-- 新版获取头像昵称（推荐） -->
<button class="avatar-btn" open-type="chooseAvatar" bindchooseavatar="onChooseAvatar">
  <image src="{{avatarUrl || '/images/default-avatar.png'}}" />
</button>
<input type="nickname" placeholder="请输入昵称" bindinput="onNicknameInput" />
```

> **Java开发者注意点**：小程序的 `button` 与 HTML 的 `<button>` 最大的区别在于 `open-type` 属性。通过 `open-type` 可以调用微信的开放能力（如获取用户信息、获取手机号、分享等），这些能力需要用户主动触发，不能通过代码自动调用 —— 这与 Web 端 OAuth2 授权流程中用户点击"同意授权"的交互逻辑一致。

#### input 与 textarea

```xml
<!-- 搜索输入框 -->
<view class="search-box">
  <input
    type="text"
    placeholder="搜索文章..."
    value="{{searchKeyword}}"
    bindinput="onSearchInput"
    bindconfirm="onSearchConfirm"
    confirm-type="search"
  />
</view>

<!-- 评论输入框 -->
<textarea
  placeholder="写下你的评论..."
  value="{{commentContent}}"
  bindinput="onCommentInput"
  maxlength="500"
  auto-height
  show-confirm-bar="{{false}}"
/>
<button type="primary" size="mini" bindtap="submitComment">发表评论</button>
```

```javascript
Page({
  data: {
    searchKeyword: '',
    commentContent: ''
  },

  onSearchInput(e) {
    this.setData({ searchKeyword: e.detail.value })
  },

  onSearchConfirm() {
    const keyword = this.data.searchKeyword.trim()
    if (!keyword) return
    wx.navigateTo({ url: `/pages/search/result?keyword=${encodeURIComponent(keyword)}` })
  },

  onCommentInput(e) {
    this.setData({ commentContent: e.detail.value })
  },

  async submitComment() {
    const content = this.data.commentContent.trim()
    if (!content) {
      wx.showToast({ title: '请输入评论内容', icon: 'none' })
      return
    }
    await wx.request({
      url: 'https://api.example.com/comments',
      method: 'POST',
      data: { articleId: '101', content }
    })
    wx.showToast({ title: '评论成功', icon: 'success' })
    this.setData({ commentContent: '' })
  }
})
```

### 9.4 媒体组件

#### image —— 图片

```xml
<!-- 文章封面图 -->
<image
  src="{{article.coverUrl}}"
  mode="aspectFill"
  class="cover-image"
  lazy-load
  bindtap="previewImage"
  data-src="{{article.coverUrl}}"
/>

<!-- 文章内容中的图片列表 -->
<image
  wx:for="{{article.images}}"
  wx:key="*this"
  src="{{item}}"
  mode="widthFix"
  class="content-image"
  lazy-load
  bindtap="previewImage"
  data-src="{{item}}"
  data-urls="{{article.images}}"
/>
```

```javascript
Page({
  previewImage(e) {
    const current = e.currentTarget.dataset.src
    const urls = e.currentTarget.dataset.urls || [current]
    wx.previewImage({ current, urls })
  }
})
```

> **Java开发者注意点**：`image` 组件的 `mode` 属性非常重要，它决定了图片的裁剪和缩放方式。常用值包括：
> - `scaleToFill`（默认）：拉伸填充，可能变形
> - `aspectFit`：等比缩放，完整显示，可能有留白
> - `aspectFill`：等比缩放，填满容器，可能裁剪
> - `widthFix`：宽度不变，高度自适应（适合文章内容图片）
>
> 这与 Java 后端使用 Thumbnailator 或 ImgScaler 处理图片缩放的逻辑类似，但小程序是在**客户端**实时处理。

### 9.5 导航组件

#### navigator —— 页面链接

```xml
<!-- 普通跳转 -->
<navigator url="/pages/article/detail?id=101" hover-class="navigator-hover">
  <text>查看文章详情</text>
</navigator>

<!-- 跳转到 TabBar 页面 -->
<navigator url="/pages/index/index" open-type="switchTab">
  <text>返回首页</text>
</navigator>

<!-- 重定向（替换当前页面） -->
<navigator url="/pages/login/login" open-type="redirect">
  <text>请先登录</text>
</navigator>
```

> **类比理解**：`navigator` 的 `open-type` 与 Spring MVC 中的 `redirect:/` 和 `forward:/` 概念类似：
> - `navigate`（默认）= `forward`，保留当前页面在页面栈中
> - `redirect` = `redirect`，替换当前页面，不可返回
> - `switchTab` = 切换到 TabBar 页面，会关闭其他所有非 TabBar 页面
> - `reLaunch` = 关闭所有页面，打开目标页面（类似重新启动应用）

### 9.6 内容资讯类场景组件选型总结

| 场景 | 推荐组件 | 关键属性 |
|------|---------|---------|
| 首页轮播图 | `swiper` + `swiper-item` | `autoplay`、`circular`、`indicator-dots` |
| 文章列表滚动 | `scroll-view` | `scroll-y`、`refresher-enabled`、`bindscrolltolower` |
| 文章详情渲染 | `rich-text` | `nodes`（HTML字符串） |
| 搜索输入 | `input` | `confirm-type="search"`、`bindconfirm` |
| 图片预览 | `image` + `wx.previewImage` | `mode="aspectFill"`、`lazy-load` |
| 分类筛选 | `picker` | `mode="selector"`、`range-key` |

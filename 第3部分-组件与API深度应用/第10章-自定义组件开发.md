## 第10章 自定义组件开发

自定义组件是小程序实现 UI 复用的核心机制。对于 Java 开发者来说，可以将自定义组件理解为 **Spring 中的自定义 Bean** 或 **Thymeleaf 的自定义 Fragment** —— 它封装了独立的 UI 逻辑和样式，可以在不同页面中复用。

### 10.1 组件创建与注册

在项目根目录下创建 `components` 文件夹，每个组件是一个独立的目录，包含四个文件：

```
components/
  news-card/
    news-card.js        # 组件逻辑
    news-card.json      # 组件配置
    news-card.wxml      # 组件模板
    news-card.wxss      # 组件样式
```

**组件配置文件（声明为自定义组件）：**

```json
// components/news-card/news-card.json
{
  "component": true,
  "usingComponents": {}
}
```

**组件逻辑文件（Component 构造器）：**

```javascript
// components/news-card/news-card.js
Component({
  // 组件属性（外部传入的数据）
  properties: {
    article: {
      type: Object,
      value: {}
    },
    showSummary: {
      type: Boolean,
      value: true
    }
  },

  // 组件内部数据
  data: {
    liked: false,
    likeCount: 0
  },

  // 组件方法
  methods: {
    onTapCard() {
      const id = this.data.article.id
      this.triggerEvent('cardtap', { articleId: id })
    },

    onLike() {
      const liked = !this.data.liked
      const likeCount = liked
        ? this.data.likeCount + 1
        : this.data.likeCount - 1
      this.setData({ liked, likeCount })
      this.triggerEvent('likechange', { liked, likeCount })
    },

    onShare() {
      this.triggerEvent('share', { article: this.data.article })
    }
  }
})
```

**组件模板文件：**

```xml
<!-- components/news-card/news-card.wxml -->
<view class="news-card" bindtap="onTapCard">
  <!-- 封面图 -->
  <image
    class="card-cover"
    src="{{article.coverUrl}}"
    mode="aspectFill"
    wx:if="{{article.coverUrl}}"
  />

  <!-- 内容区 -->
  <view class="card-body">
    <text class="card-title">{{article.title}}</text>

    <!-- 摘要（通过 slot 支持自定义） -->
    <view class="card-summary" wx:if="{{showSummary}}">
      <text class="summary-text">{{article.summary}}</text>
    </view>

    <!-- 底部信息栏 -->
    <view class="card-footer">
      <view class="author-info">
        <image class="avatar" src="{{article.authorAvatar}}" />
        <text class="author-name">{{article.authorName}}</text>
        <text class="publish-time">{{article.publishTime}}</text>
      </view>
      <view class="action-bar">
        <view class="action-item" catchtap="onLike">
          <text class="action-icon {{liked ? 'liked' : ''}}">
            {{liked ? '❤' : '♡'}}
          </text>
          <text class="action-count">{{likeCount || article.likeCount}}</text>
        </view>
        <view class="action-item" catchtap="onShare">
          <text class="action-icon">↗</text>
        </view>
      </view>
    </view>
  </view>

  <!-- 默认插槽：允许父组件插入额外内容 -->
  <slot name="extra"></slot>
</view>
```

### 10.2 组件属性 properties

> **类比理解**：`properties` 就像 Java POJO 的属性 + Spring MVC 的 `@ModelAttribute`。它是组件对外的"接口"，定义了父组件可以向子组件传递哪些数据。

```javascript
Component({
  properties: {
    // 简写形式
    title: String,

    // 完整定义形式
    article: {
      type: Object,          // 类型：String, Number, Boolean, Object, Array, null(任意类型)
      value: {},             // 默认值
      observer: function(newVal, oldVal) {
        // 属性值变化时的回调（类似 Vue 的 watch）
        console.log('article 属性变化', oldVal, '->', newVal)
      }
    },

    // 自定义校验
    pageSize: {
      type: Number,
      value: 10,
      observer(newVal) {
        if (newVal < 1 || newVal > 100) {
          console.warn('pageSize 应在 1-100 之间')
        }
      }
    }
  }
})
```

> **Java开发者注意点**：小程序的 `properties` 与 Java 的类属性有本质区别：
> 1. **类型系统不同**：Java 是强类型编译期检查，小程序 `properties` 的 `type` 仅做运行时类型提示，不会阻止错误类型赋值。
> 2. **observer 模式**：小程序的 `observer` 类似于 Java 的 `PropertyChangeListener`，但更轻量，直接在属性定义中声明。
> 3. **不可变性**：`properties` 的值在组件内部可以被修改（通过 `this.setData`），但建议将其视为只读，遵循单向数据流原则。

### 10.3 组件生命周期

```javascript
Component({
  lifetimes: {
    // 组件实例刚被创建好（此时还不能调用 setData）
    created() {
      console.log('组件创建 —— 类比 Java 构造函数')
      // 适合做初始化工作，但不能操作 DOM/WXML
    },

    // 组件实例进入页面节点树（可以操作节点，但尚未完成布局）
    attached() {
      console.log('组件挂载 —— 类比 Spring Bean 的 @PostConstruct')
      // 适合做需要访问节点信息的初始化
    },

    // 组件在视图层布局完成后执行
    ready() {
      console.log('组件就绪 —— 类比 Java Swing 的 setVisible(true) 之后')
      // 适合获取节点尺寸、位置等信息
      const query = this.createSelectorQuery()
      query.select('.news-card').boundingClientRect(rect => {
        console.log('卡片高度：', rect.height)
      }).exec()
    },

    // 组件实例被从页面节点树移除
    detached() {
      console.log('组件卸载 —— 类比 Spring Bean 的 @PreDestroy')
      // 适合做清理工作，如清除定时器
      if (this._timer) {
        clearInterval(this._timer)
      }
    },

    // 组件抛出错误时执行
    error(err) {
      console.error('组件错误：', err)
    }
  },

  // 组件所在页面的生命周期（可选）
  pageLifetimes: {
    show() {
      // 组件所在页面显示时触发
      console.log('所在页面显示')
    },
    hide() {
      // 组件所在页面隐藏时触发
    },
    resize(size) {
      // 组件所在页面尺寸变化时触发
      console.log('页面尺寸变化：', size)
    }
  }
})
```

> **类比理解**：组件生命周期与 Spring Bean 生命周期的对应关系：
>
> | 小程序组件 | Spring Bean | 说明 |
> |-----------|-------------|------|
> | `created` | 构造函数 | 实例化，但尚未完成依赖注入 |
> | `attached` | `@PostConstruct` | 属性注入完成，可以使用依赖 |
> | `ready` | `afterPropertiesSet` | 所有初始化完成，可以正常工作 |
> | `detached` | `@PreDestroy` | 销毁前清理资源 |

### 10.4 组件间通信

#### 父传子：通过 properties

```xml
<!-- 父页面使用子组件 -->
<news-card article="{{currentArticle}}" show-summary="{{true}}" />
```

#### 子传父：通过 triggerEvent

```javascript
// 子组件内部触发事件
Component({
  methods: {
    onLike() {
      this.triggerEvent('likechange', {
        liked: true,
        articleId: this.data.article.id
      }, { bubbles: true, composed: true })
    }
  }
})
```

```xml
<!-- 父组件监听子组件事件 -->
<news-card
  article="{{article}}"
  bind:likechange="onArticleLike"
  bind:cardtap="onCardTap"
/>
```

```javascript
// 父组件的事件处理函数
Page({
  onArticleLike(e) {
    const { liked, articleId } = e.detail
    console.log(`文章 ${articleId} 被${liked ? '点赞' : '取消点赞'}`)
    // 调用后端 API 更新点赞状态
    wx.request({
      url: `https://api.example.com/articles/${articleId}/like`,
      method: 'POST',
      data: { liked }
    })
  },

  onCardTap(e) {
    const { articleId } = e.detail
    wx.navigateTo({ url: `/pages/article/detail?id=${articleId}` })
  }
})
```

> **类比理解**：`triggerEvent` 的机制与 Java 中的**观察者模式**（Observer Pattern）非常相似：
> - 子组件 = 被观察者（Event Publisher）
> - 父组件 = 观察者（Event Subscriber）
> - `triggerEvent` = `eventBus.publish(event)`
> - `bind:eventName` = `@EventListener`
>
> 选项参数 `{ bubbles: true, composed: true }` 类似于 DOM 事件冒泡机制。

### 10.5 组件插槽 slot 机制

```xml
<!-- 子组件模板：定义插槽 -->
<view class="card-wrapper">
  <view class="card-header">
    <!-- 默认插槽 -->
    <slot></slot>
  </view>
  <view class="card-body">
    <text>{{content}}</text>
  </view>
  <view class="card-footer">
    <!-- 具名插槽 -->
    <slot name="actions"></slot>
  </view>
</view>
```

```xml
<!-- 父组件使用插槽 -->
<news-card article="{{article}}">
  <!-- 填充默认插槽 -->
  <view class="custom-header">
    <text class="tag">热门</text>
  </view>

  <!-- 填充具名插槽 -->
  <view slot="actions">
    <button size="mini" bindtap="onBookmark">收藏</button>
    <button size="mini" bindtap="onShare">分享</button>
  </view>
</news-card>
```

> **Java开发者注意点**：`slot` 机制与 Thymeleaf 的 `th:fragment` / `th:replace` 概念类似，都是"模板中的占位符，由调用方填充内容"。在 Java Web 中，这类似于：
> ```html
> <!-- Thymeleaf 布局 -->
> <div th:replace="~{fragments/header :: custom-header}"></div>
> ```

### 10.6 完整实战：新闻卡片组件

以下是在页面中使用新闻卡片组件的完整示例。

**页面配置（注册组件）：**

```json
// pages/index/index.json
{
  "usingComponents": {
    "news-card": "/components/news-card/news-card"
  }
}
```

**页面模板：**

```xml
<!-- pages/index/index.wxml -->
<view class="page-container">
  <!-- 顶部搜索栏 -->
  <view class="search-bar">
    <input
      class="search-input"
      placeholder="搜索文章..."
      bindconfirm="onSearch"
    />
  </view>

  <!-- 分类标签栏 -->
  <scroll-view scroll-x class="category-bar">
    <view
      wx:for="{{categories}}"
      wx:key="id"
      class="category-tag {{activeCategory === item.id ? 'active' : ''}}"
      bindtap="onCategoryTap"
      data-id="{{item.id}}"
    >
      {{item.name}}
    </view>
  </scroll-view>

  <!-- 文章列表 -->
  <scroll-view
    scroll-y
    class="article-list"
    refresher-enabled
    bindrefresherrefresh="onRefresh"
    refresher-triggered="{{refreshing}}"
    bindscrolltolower="loadMore"
  >
    <news-card
      wx:for="{{articles}}"
      wx:key="id"
      article="{{item}}"
      show-summary="{{true}}"
      bind:cardtap="onCardTap"
      bind:likechange="onLikeChange"
    />
    <view wx:if="{{!hasMore && articles.length > 0}}" class="no-more">
      —— 没有更多了 ——
    </view>
    <view wx:if="{{articles.length === 0 && !loading}}" class="empty">
      暂无文章
    </view>
  </scroll-view>
</view>
```

**页面逻辑：**

```javascript
// pages/index/index.js
import { request } from '../../utils/request'

Page({
  data: {
    categories: [],
    activeCategory: 0,
    articles: [],
    page: 1,
    refreshing: false,
    loading: false,
    hasMore: true
  },

  onLoad() {
    this.loadCategories()
    this.loadArticles(true)
  },

  async loadCategories() {
    const res = await request({ url: '/api/categories' })
    this.setData({ categories: res.data })
  },

  async loadArticles(isRefresh) {
    if (this.data.loading) return
    this.setData({
      loading: true,
      refreshing: isRefresh ? true : this.data.refreshing,
      page: isRefresh ? 1 : this.data.page
    })

    try {
      const res = await request({
        url: '/api/articles',
        data: {
          categoryId: this.data.activeCategory,
          page: this.data.page,
          size: 10
        }
      })

      const newArticles = res.data.content
      this.setData({
        articles: isRefresh ? newArticles : this.data.articles.concat(newArticles),
        hasMore: !res.data.last,
        page: isRefresh ? 1 : this.data.page
      })
    } finally {
      this.setData({ loading: false, refreshing: false })
    }
  },

  onRefresh() {
    this.loadArticles(true)
  },

  loadMore() {
    if (this.data.hasMore) {
      this.setData({ page: this.data.page + 1 })
      this.loadArticles(false)
    }
  },

  onCategoryTap(e) {
    const id = e.currentTarget.dataset.id
    this.setData({ activeCategory: id, articles: [], page: 1, hasMore: true })
    this.loadArticles(true)
  },

  onCardTap(e) {
    wx.navigateTo({ url: `/pages/article/detail?id=${e.detail.articleId}` })
  },

  async onLikeChange(e) {
    const { liked, articleId } = e.detail
    await request({
      url: `/api/articles/${articleId}/like`,
      method: 'POST',
      data: { liked }
    })
  },

  onSearch(e) {
    const keyword = e.detail.value.trim()
    if (keyword) {
      wx.navigateTo({
        url: `/pages/search/result?keyword=${encodeURIComponent(keyword)}`
      })
    }
  }
})
```

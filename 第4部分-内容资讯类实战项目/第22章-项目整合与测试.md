> **Java开发者注意点**：联调阶段最常见的坑是 **HTTPS 和域名白名单**。小程序正式环境只允许请求已配置的 HTTPS 域名。开发阶段可以在开发者工具中勾选"不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书"，但真机预览时需要确保后端已部署到 HTTPS 环境。

### 22.2 MyBatis-Plus 分页插件配置

```java
package com.example.newsapp.config;

import com.baomidou.mybatisplus.annotation.DbType;
import com.baomidou.mybatisplus.extension.plugins.MybatisPlusInterceptor;
import com.baomidou.mybatisplus.extension.plugins.inner.PaginationInnerInterceptor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class MybatisPlusConfig {

    /**
     * 分页插件 —— 必须配置，否则 Page 分页不生效
     */
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        interceptor.addInnerInterceptor(
            new PaginationInnerInterceptor(DbType.MYSQL)
        );
        return interceptor;
    }
}
```

### 22.4 常见Bug排查清单

| 问题 | 可能原因 | 排查方法 |
|------|---------|---------|
| 页面白屏 | JS报错、数据未加载 | 查看Console面板、检查Network |
| 数据不显示 | 字段名不匹配 | 对比后端返回字段与前端绑定字段 |
| 登录失败 | appid/secret错误 | 检查application.yml配置 |
| Token过期 | JWT过期时间太短 | 检查JwtUtil的expiration配置 |
| 图片不显示 | 域名未配置白名单 | 检查downloadFile合法域名 |
| 上拉加载重复 | page未正确递增 | 检查loadArticles中page逻辑 |
| 收藏状态不同步 | 本地缓存与服务器不一致 | 每次进入详情页重新checkFavorite |
| 分享卡片无图 | imageUrl为空或域名不合法 | 确保coverUrl为https完整URL |
| 夜间模式闪烁 | 切换时未同步导航栏 | 调用wx.setNavigationBarColor |
| iOS样式异常 | rpx计算差异 | 使用安全区域适配env() |

> **Java开发者注意点**：小程序调试与 Java Web 调试有相似之处，也有独特之处：
> - **相似**：都可以通过 Network 面板查看请求/响应、通过 Console 查看日志
> - **不同**：小程序没有浏览器 DevTools 的 Elements 面板来实时修改 DOM，但提供了 **Wxml Panel** 和 **AppData Panel** 来查看和修改页面数据
> - **技巧**：在 `request.js` 中添加 `console.log('请求:', options.url, '参数:', options.data)` 可以快速定位接口调用问题

### 22.5 体验评分优化建议

微信开发者工具内置了"体验评分"功能，以下是一些常见优化项：

```json
// app.json —— 性能优化配置
{
  "lazyCodeLoading": "requiredComponents",
  "renderer": "skyline",
  "componentFramework": "glass-easel"
}
```

```javascript
// 优化建议清单

// 1. 图片懒加载：所有image标签添加 lazy-load 属性
// <image lazy-load src="{{item.coverUrl}}" />

// 2. setData优化：避免频繁调用setData，合并数据更新
// 差：多次setData
this.setData({ title: '...' })
this.setData({ content: '...' })
// 好：一次setData
this.setData({ title: '...', content: '...' })

// 3. 避免setData传输大量数据
// 差：setData整个大数组
this.setData({ articles: hugeArray })
// 好：只setData变化的部分
this.setData({ ['articles[' + index + '].liked']: true })

// 4. 使用自定义组件实现局部更新
// 将频繁变化的UI封装为Component，利用组件的setData作用域

// 5. 分包加载
// app.json
{
  "subpackages": [
    {
      "root": "pages/profile",
      "pages": ["profile", "favorites", "history"]
    }
  ]
}
```

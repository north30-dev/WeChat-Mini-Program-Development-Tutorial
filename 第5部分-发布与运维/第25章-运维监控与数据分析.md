### 25.3 实时日志与异常上报

使用 `wx.getRealtimeLogManager` 收集实时日志：

```javascript
// utils/logger.js
const logger = wx.getRealtimeLogManager()

// 封装日志工具
const log = {
  info(msg, data) {
    logger.info(msg, data)
    console.log(`[INFO] ${msg}`, data)
  },

  warn(msg, data) {
    logger.warn(msg, data)
    console.warn(`[WARN] ${msg}`, data)
  },

  error(msg, data) {
    logger.error(msg, data)
    console.error(`[ERROR] ${msg}`, data)
    // 同时上报到自有服务端
    this.reportToServer('error', msg, data)
  },

  // 上报到自有后端（与 Java 服务对接）
  reportToServer(level, msg, data) {
    wx.request({
      url: `${getApp().globalData.baseUrl}/api/log/report`,
      method: 'POST',
      data: {
        level,
        message: msg,
        data: JSON.stringify(data),
        page: getCurrentPages()[getCurrentPages().length - 1]?.route || '',
        timestamp: Date.now(),
        systemInfo: wx.getSystemInfoSync()
      }
    })
  }
}

module.exports = log
```

**使用示例**：

```javascript
const logger = require('../../utils/logger')

Page({
  onLoad() {
    logger.info('页面加载', { page: 'pages/index/index' })
  },

  async fetchData() {
    try {
      const res = await wx.request({ url: '/api/data' })
      logger.info('数据请求成功', { url: '/api/data', status: res.statusCode })
    } catch (err) {
      logger.error('数据请求失败', { url: '/api/data', error: err.message, stack: err.stack })
    }
  }
})
```

> **Java 开发者注意点：** `wx.getRealtimeLogManager` 类似于 Java 中的 Logback/Log4j2，但日志是上报到微信服务器的。在微信公众平台 → **开发管理** → **运维中心** → **实时日志** 中可以按用户 OpenID 过滤查看。对于需要深度分析的场景，建议同时上报到自有后端，配合 ELK（Elasticsearch + Logstash + Kibana）或 Loki 等工具进行集中管理。

### 25.4 性能监控指标解读

在微信公众平台 → **运维中心** → **性能数据** 中查看核心性能指标：

| 指标 | 全称 | 含义 | 优秀标准 |
|------|------|------|---------|
| FCP | First Contentful Paint | 首次内容绘制 | < 1.5s |
| LCP | Largest Contentful Paint | 最大内容绘制 | < 2.5s |
| FID | First Input Delay | 首次输入延迟 | < 100ms |
| FMP | First Meaningful Paint | 首次有意义的绘制 | < 2.0s |
| TTI | Time to Interactive | 可交互时间 | < 3.0s |

> **类比理解：** 这些指标与 Java Web 前端性能监控中的 Core Web Vitals 概念一致。如果你的 Spring Boot 应用有前端页面，同样会关注这些指标。小程序的优势在于运行在微信客户端沙箱中，由平台统一进行性能度量。

### 25.5 用户行为分析

在微信公众平台 → **统计** → **访问分析** 中可以查看：

- **页面访问路径**：用户从哪个页面进入，经过哪些页面，最终在哪里离开
- **页面热度**：各页面的访问量和访问时长排名
- **用户画像**：性别、年龄、地区、设备分布
- **留存分析**：次日留存、7日留存、30日留存趋势

**自定义埋点示例**：

```javascript
// utils/track.js
const track = {
  // 页面访问埋点
  pageView(pageName, params = {}) {
    wx.reportAnalytics('page_view', {
      page_name: pageName,
      ...params,
      timestamp: Date.now()
    })
  },

  // 点击事件埋点
  clickEvent(eventName, params = {}) {
    wx.reportAnalytics('click_event', {
      event_name: eventName,
      ...params,
      timestamp: Date.now()
    })
  },

  // 自定义业务事件
  businessEvent(eventName, data = {}) {
    wx.reportAnalytics(eventName, data)
  }
}

module.exports = track
```

> **Java 开发者注意点：** `wx.reportAnalytics` 类似于 Java Web 中集成 Google Analytics / 百度统计 / 神策数据等第三方分析 SDK 的埋点能力。不同之处在于，小程序的埋点数据由微信平台托管，在后台直接查看，无需自行搭建数据采集管道。

### 25.6 与 Java Web 运维工具对比

| 能力 | 微信小程序 | Java Web |
|------|-----------|----------|
| 日志收集 | `wx.getRealtimeLogManager` + 自建上报 | Logback → ELK / Loki |
| 性能监控 | 微信运维中心（平台内置） | Prometheus + Grafana |
| 错误追踪 | 微信后台异常监控 | Sentry / SkyWalking |
| 数据分析 | 微信统计 + 数据助手 | Google Analytics / 神策 |
| 告警通知 | 微信后台消息通知 | Prometheus AlertManager / 钉钉机器人 |
| 链路追踪 | 有限支持（仅请求级别） | Zipkin / Jaeger |

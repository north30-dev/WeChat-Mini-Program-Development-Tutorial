# 微信小程序开发入门教程

> **适用读者**：具有 Java Web 开发经验（Spring Boot / Spring MVC）的后端开发者
> **学习目标**：从零掌握微信小程序开发，能够独立完成内容资讯类小程序项目
> **教程特色**：Java 视角对比、完整实战项目、企业级开发规范

## 第23章 小程序发布流程

### 23.1 代码上传与版本管理

在微信开发者工具中完成开发后，点击工具栏的 **上传** 按钮，填写版本号和项目备注即可将代码提交至微信后台。

| 操作 | 微信小程序 | Java Web（类比） |
|------|-----------|-----------------|
| 代码提交 | 开发者工具 → 上传 | `git push` 推送代码 |
| 版本标识 | 版本号 + 备注 | Git Tag / Commit Hash |
| 部署环境 | 后台选择发布 | CI/CD 流水线部署 |
| 回滚方式 | 后台版本回退 | 重新部署旧版本镜像/JAR |

> **类比理解：** 微信的"上传"类似于 `git push origin main`，但代码是推送到微信服务器而非 Git 仓库。每次上传会生成一个"开发版本"，需经过审核后才能成为"线上版本"。

### 23.2 提交审核注意事项

上传成功后，在 [微信公众平台](https://mp.weixin.qq.com) → **版本管理** 中找到刚上传的版本，点击 **提交审核**：

1. **填写功能页面**：选择小程序中包含的核心功能页面路径
2. **配置测试账号**：如小程序需要登录，必须提供可用的测试账号和密码
3. **等待审核**：通常 1-7 个工作日，首次提交可能更久

### 23.3 审核常见被拒原因与解决方案（完整清单）

| 被拒原因 | 说明 | 解决方案 |
|---------|------|---------|
| 类目与功能不符 | 所选服务类目与实际功能不一致 | 重新选择匹配的类目，必要时补充资质 |
| 缺少用户隐私协议 | 未配置或隐私协议内容不完整 | 在 `app.json` 中配置 `"__usePrivacyCheck__": true`，并完善隐私保护指引 |
| 测试账号无法登录 | 提供的测试账号过期或密码错误 | 确保测试账号在审核期间有效且可正常使用 |
| 内容违规 | 含有敏感词、虚假宣传、侵权内容 | 全面自查文案和图片，确保合规 |
| 个人信息收集不合规 | 未明示收集目的或超范围收集 | 遵循最小必要原则，在隐私协议中逐项说明 |
| 空白页面/功能未完成 | 存在未实现的页面或占位内容 | 确保所有页面功能完整可用 |
| 引导关注/外链 | 包含引导关注公众号或跳转外部链接的入口 | 移除所有违规引导内容 |
| 体验不佳 | 页面加载慢、交互异常、UI 错乱 | 充分测试后再提交，关注低端机型适配 |

### 23.4 正式发布与灰度发布

审核通过后，版本状态变为 **"审核通过"**，此时可以：

- **全量发布**：点击"发布"按钮，所有用户立即获取新版本
- **灰度发布**：点击"灰度发布"，按比例逐步推送给用户

```
灰度发布配置示例：
- 第一阶段：5% 用户，观察 24 小时
- 第二阶段：20% 用户，观察 24 小时
- 第三阶段：50% 用户，观察 24 小时
- 第四阶段：100% 全量发布
```

> **Java 开发者注意点：** 灰度发布在 Java Web 中通常通过 Nginx 灰度路由、Kubernetes Rolling Update 或 Istio 流量拆分实现。微信小程序的灰度发布由平台原生支持，无需自行搭建基础设施，这大大降低了运维复杂度。

### 23.5 审核避坑清单

- [ ] **类目选择与功能匹配**：确认所选类目与小程序核心功能一致，特殊类目需提供相应资质（如食品、医疗等）
- [ ] **用户隐私协议配置**：在 `app.json` 中启用隐私检查，确保所有涉及个人信息收集的页面都有弹窗授权
- [ ] **测试账号提供**：如需登录，提供至少一个有效测试账号；如需付费，提供测试支付环境
- [ ] **内容合规性检查**：自查所有文案、图片、视频，确保不含敏感内容
- [ ] **个人信息收集合规**：遵循"最小必要"原则，收集前明确告知用户目的和范围
- [ ] **功能完整性**：确保所有入口均可正常访问，无空白页和占位内容
- [ ] **隐私接口声明**：在 `app.json` 的 `requiredPrivateInfos` 中声明使用的隐私接口（如 `getLocation`、`chooseAddress`）

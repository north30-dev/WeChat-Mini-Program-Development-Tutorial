# 微信小程序开发入门教程

> **适用读者**：具有 Java Web 开发经验（Spring Boot / Spring MVC）的后端开发者
> **学习目标**：从零掌握微信小程序开发，能够独立完成内容资讯类小程序项目
> **教程特色**：Java 视角对比、完整实战项目、企业级开发规范

---

## 附录D：审核规范 Checklist

### D.1 基础合规检查项

| 序号 | 检查项 | 要求说明 | 状态 |
|------|--------|----------|------|
| 1 | 服务类目选择 | 选择与小程序功能匹配的类目，部分类目需要提供资质证明 | [ ] |
| 2 | 主体资质 | 个人主体或企业主体，企业需提供营业执照 | [ ] |
| 3 | 隐私保护指引 | 必须填写《小程序用户隐私保护指引》，声明收集的用户信息及用途 | [ ] |
| 4 | 用户隐私协议 | 首次使用时需弹出隐私协议弹窗，用户同意后方可使用 | [ ] |
| 5 | 隐私接口声明 | 在 app.json 中声明 `__usePrivacyCheck__: true` | [ ] |
| 6 | 实名认证 | 小程序管理员需完成微信实名认证 | [ ] |
| 7 | 基本信息完善 | 小程序名称、图标、简介、服务类目等基本信息完整 | [ ] |

### D.2 功能完整性检查项

| 序号 | 检查项 | 要求说明 | 状态 |
|------|--------|----------|------|
| 1 | 核心功能可用 | 所有页面和功能可正常访问和操作，无空白页、无死链 | [ ] |
| 2 | 数据加载正常 | 列表数据可正常加载，分页功能正常 | [ ] |
| 3 | 表单提交正常 | 表单校验完善，提交后数据正确保存 | [ ] |
| 4 | 图片/文件上传 | 图片上传功能正常，支持预览 | [ ] |
| 5 | 登录/注册流程 | 登录注册流程完整，异常情况有友好提示 | [ ] |
| 6 | 网络异常处理 | 无网络或弱网环境下有友好提示，不出现白屏 | [ ] |
| 7 | 空状态处理 | 列表为空时有空状态提示（非空白页） | [ ] |
| 8 | 加载状态 | 数据加载时有 loading 提示 | [ ] |
| 9 | 下拉刷新/上拉加载 | 列表页支持下拉刷新和上拉加载更多 | [ ] |
| 10 | 返回导航 | 每个页面都有明确的返回路径 | [ ] |

### D.3 UI/UX 规范检查项

| 序号 | 检查项 | 要求说明 | 状态 |
|------|--------|----------|------|
| 1 | 导航栏标题 | 每个页面都有清晰的导航栏标题 | [ ] |
| 2 | 底部安全区域 | 页面底部内容不被手机底部横条遮挡（使用 `safe-area-inset-bottom`） | [ ] |
| 3 | 顶部安全区域 | 刘海屏适配，内容不被状态栏/刘海遮挡 | [ ] |
| 4 | 字体大小 | 正文不小于 24rpx，标题层级分明 | [ ] |
| 5 | 按钮可点击区域 | 按钮点击区域不小于 44px（88rpx） | [ ] |
| 6 | 图片适配 | 图片在不同屏幕尺寸下显示正常，无变形 | [ ] |
| 7 | 深色模式 | 如支持深色模式，确保所有页面适配 | [ ] |
| 8 | 横屏适配 | 如支持横屏，确保布局正确 | [ ] |
| 9 | 键盘弹起适配 | 输入框聚焦时，页面不被键盘遮挡 | [ ] |
| 10 | 动画流畅 | 页面切换和交互动画流畅，无卡顿 | [ ] |

### D.4 内容安全检查项

| 序号 | 检查项 | 要求说明 | 状态 |
|------|--------|----------|------|
| 1 | 文字内容合规 | 无违法违规、色情暴力、政治敏感内容 | [ ] |
| 2 | 图片内容合规 | 图片无违规内容 | [ ] |
| 3 | 用户生成内容 | UGC 内容需接入微信内容安全检测 API（`security.msgSecCheck`、`security.imgSecCheck`） | [ ] |
| 4 | 虚假宣传 | 无虚假宣传、过度承诺 | [ ] |
| 5 | 知识产权 | 无侵权内容（图片、字体、音乐等有合法授权） | [ ] |
| 6 | 个人信息保护 | 不收集与服务无关的用户信息 | [ ] |
| 7 | 敏感信息展示 | 手机号、身份证等敏感信息需脱敏展示 | [ ] |
| 8 | 外部链接 | 不包含未经审核的外部链接或二维码 | [ ] |
| 9 | 分享内容 | 分享的标题、图片内容合规 | [ ] |
| 10 | 客服功能 | 提供有效的客服联系方式 | [ ] |

### D.5 技术规范检查项

| 序号 | 检查项 | 要求说明 | 状态 |
|------|--------|----------|------|
| 1 | HTTPS 接口 | 所有后端接口必须使用 HTTPS 协议 | [ ] |
| 2 | 域名白名单 | 在小程序管理后台配置 request、uploadFile、downloadFile 等域名白名单 | [ ] |
| 3 | 域名备案 | 域名需完成 ICP 备案 | [ ] |
| 4 | 服务器配置 | 服务器需配置 SSL 证书，TLS 版本 >= 1.2 | [ ] |
| 5 | 小程序包大小 | 主包不超过 2MB，总包不超过 20MB（使用分包加载） | [ ] |
| 6 | 基础库版本 | 设置合理的基础库最低版本（建议不低于 2.25.0） | [ ] |
| 7 | AppID 配置 | 使用正式 AppID，不使用测试号 | [ ] |
| 8 | 体验版测试 | 体验版功能测试完整，无严重 Bug | [ ] |
| 9 | setData 性能 | 避免频繁调用 setData，单次传输数据不超过 1MB | [ ] |
| 10 | 废弃 API | 不使用已废弃的 API（如 `wx.getUserInfo`） | [ ] |

## 第3章 小程序技术架构深度解析

### 3.1 双线程架构：逻辑层与视图层

微信小程序采用**双线程架构**，这是理解小程序运行机制的关键。

```
┌─────────────────────────────────────────────────────────────────┐
│                         微信客户端                                │
│                                                                   │
│  ┌────────────────────────┐    ┌────────────────────────┐       │
│  │     逻辑层（JSCore）     │    │     视图层（WebView）     │       │
│  │                         │◄──►│                         │       │
│  │  • 业务逻辑处理          │    │  • WXML 模板渲染         │       │
│  │  • 数据状态管理          │    │  • WXSS 样式应用         │       │
│  │  • 网络请求 wx.request   │    │  • 用户交互响应          │       │
│  │  • 微信 API 调用         │    │  • 事件监听处理          │       │
│  │                         │    │  • 每个页面独立 WebView   │       │
│  └────────────┬───────────┘    └────────────┬───────────┘       │
│               │                               │                   │
│               └──────────────┬────────────────┘                   │
│                              │                                    │
│               ┌──────────────▼────────────────┐                   │
│               │      Native 层（微信原生模块）   │                   │
│               │  线程间通信桥接 | 原生能力调用    │                   │
│               └───────────────────────────────┘                   │
│                                                                   │
│  类比：逻辑层 ≈ Spring Boot 后端                                  │
│        视图层 ≈ Vue/React 前端                                    │
│        Native层 ≈ Servlet 容器                                     │
└─────────────────────────────────────────────────────────────────┘
```

**双线程的核心要点：**

1. **逻辑层（JSCore）**：运行在独立的 JavaScript 引擎中，负责业务逻辑、数据处理、网络请求等。它没有 DOM/BOM 对象，不能直接操作页面元素。
2. **视图层（WebView）**：运行在 WebView 中，负责页面渲染和用户交互。每个页面对应一个 WebView 实例。
3. **线程间通信**：逻辑层和视图层通过微信客户端的 Native 层进行消息转发，数据和事件在两个线程之间序列化传输。

> **类比理解**
>
> 如果把 Java Web 开发中的 **前后端分离架构** 做一个类比：
>
> | 小程序双线程 | Java 前后端分离 |
> |---|---|
> | 逻辑层（JSCore） | Spring Boot 后端（提供 API） |
> | 视图层（WebView） | 前端（Vue/React，浏览器渲染） |
> | Native 层消息转发 | HTTP 协议（JSON 数据交换） |
> | `setData()` 推送数据 | 后端返回 JSON，前端渲染 |
> | 用户事件回调 | 前端发送 HTTP 请求到后端 |
>
> 但要注意一个重要区别：Java 前后端分离是**跨网络**的，而小程序的双线程是**进程内**的，通信效率更高，但仍然需要序列化/反序列化，所以 `setData()` 传递大量数据时会有性能问题。

### 3.2 数据通信机制

小程序的数据通信流程如下：

**数据从逻辑层到视图层（数据绑定）：**

```javascript
// 逻辑层：通过 setData 将数据推送到视图层
Page({
  data: {
    userName: '张三',
    score: 95
  },

  onLoad() {
    // 模拟从后端 API 获取数据
    this.fetchUserInfo();
  },

  fetchUserInfo() {
    wx.request({
      url: 'https://api.example.com/user/info',
      method: 'GET',
      success: (res) => {
        // 关键：必须通过 setData 更新数据，视图层才会刷新
        this.setData({
          userName: res.data.name,
          score: res.data.score
        });
      }
    });
  }
})
```

```xml
<!-- 视图层：通过 {{}} 双花括号绑定数据 -->
<view class="user-card">
  <text>用户名：{{userName}}</text>
  <text>分数：{{score}}</text>
</view>
```

**数据从视图层到逻辑层（事件绑定）：**

```xml
<!-- 视图层：通过 bindtap 绑定点击事件 -->
<button bindtap="handleClick">提交</button>

<!-- 也可以通过 bindinput 绑定输入事件 -->
<input bindinput="handleInput" placeholder="请输入内容" />
```

```javascript
// 逻辑层：事件处理函数
Page({
  handleClick() {
    console.log('按钮被点击了');
  },

  handleInput(e) {
    // e.detail.value 获取输入框的值
    console.log('输入内容：', e.detail.value);
  }
})
```

> **Java 开发者注意点**
>
> | 小程序数据通信 | Java Web 对应 |
> | |---|---|
> | `this.setData({key: value})` | `model.addAttribute("key", value)`（Thymeleaf） |
> | `{{userName}}` 数据绑定 | `${userName}`（Thymeleaf 表达式） |
> | `bindtap="handleClick"` | `<form action="/submit">` 或 `@PostMapping` |
> | `e.detail.value` 获取事件数据 | `@RequestParam` 或 `@RequestBody` |
>
> **性能警告**：`setData()` 是跨线程通信，每次调用都会触发一次完整的序列化和视图层 diff 更新。不要频繁调用 `setData()`，也不要一次性传递大量数据。这类似于在 Java Web 中频繁进行数据库查询或返回超大 JSON 响应——都会影响性能。

### 3.3 小程序生命周期

小程序的生命周期分为 **App 级别**和 **Page 级别**两个层次。

**App 生命周期：**

```javascript
// app.js —— 小程序入口文件
App({
  // 小程序初始化完成时触发（全局只触发一次）
  onLaunch(options) {
    console.log('小程序启动，启动参数：', options);
    // 类比：Spring Boot 的 ApplicationRunner / CommandLineRunner
    // 适合在这里做：登录检查、全局数据初始化
    this.checkLoginStatus();
  },

  // 小程序显示（从后台进入前台）
  onShow(options) {
    console.log('小程序显示');
    // 类比：应用从最小化恢复
  },

  // 小程序隐藏（从前台进入后台）
  onHide() {
    console.log('小程序隐藏');
    // 类比：应用最小化
  },

  // 全局数据（类比 Spring 的 Application 级别的 Bean）
  globalData: {
    userInfo: null,
    token: '',
    baseUrl: 'https://api.example.com'
  },

  // 自定义方法
  checkLoginStatus() {
    const token = wx.getStorageSync('token');
    if (token) {
      this.globalData.token = token;
    }
  }
})
```

**Page 生命周期：**

```javascript
// pages/index/index.js
Page({
  data: {
    list: [],
    loading: false
  },

  // 页面加载（一个页面只调用一次）
  // 类比：Controller 的 @GetMapping 方法执行前的初始化
  onLoad(options) {
    console.log('页面加载，参数：', options);
    // options 是页面跳转时传递的参数
    // 类比：@GetMapping("/page") 中的 @RequestParam
    this.loadData();
  },

  // 页面显示（每次进入页面都会调用）
  onShow() {
    console.log('页面显示');
    // 类比：每次页面刷新时重新获取数据
  },

  // 页面初次渲染完成
  onReady() {
    console.log('页面渲染完成');
    // 类比：页面 DOM 加载完成（$(document).ready()）
  },

  // 页面隐藏（跳转到其他页面或小程序进入后台）
  onHide() {
    console.log('页面隐藏');
  },

  // 页面卸载（使用 wx.navigateBack 或 redirectTo 时触发）
  onUnload() {
    console.log('页面卸载');
    // 类比：Servlet 的 destroy() 方法
  },

  // 下拉刷新
  onPullDownRefresh() {
    console.log('用户下拉刷新');
    this.loadData().then(() => {
      wx.stopPullDownRefresh();
    });
  },

  // 上拉触底
  onReachBottom() {
    console.log('滚动到底部，加载更多');
    this.loadMore();
  },

  // 页面滚动
  onPageScroll(e) {
    // e.scrollTop 为滚动距离
  },

  // 数据加载方法
  loadData() {
    this.setData({ loading: true });
    return new Promise((resolve) => {
      wx.request({
        url: getApp().globalData.baseUrl + '/api/list',
        success: (res) => {
          this.setData({ list: res.data, loading: false });
          resolve();
        },
        fail: () => {
          this.setData({ loading: false });
          resolve();
        }
      });
    });
  }
})
```

> **类比理解：生命周期对比**
>
> | 小程序生命周期 | Java Web/Spring 对应 | 说明 |
> | |---|---|---|
> | `App.onLaunch()` | `ApplicationRunner.run()` | 应用启动时执行一次 |
> | `App.onShow()` | 应用恢复前台 | 从后台切回前台 |
> | `App.onHide()` | 应用进入后台 | 切到其他应用 |
> | `Page.onLoad()` | `@GetMapping` 方法执行 | 页面首次加载，接收路由参数 |
> | `Page.onShow()` | 页面每次可见时 | 从其他页面返回时也会触发 |
> | `Page.onReady()` | 页面渲染完成 | 首次渲染完成后触发一次 |
> | `Page.onUnload()` | Servlet `destroy()` | 页面被销毁时 |
> | `Page.onPullDownRefresh()` | 无直接对应 | 小程序特有，需在 JSON 中配置 |
>
> **关键区别**：在 Java Web 中，每个 HTTP 请求都是独立的（无状态），控制器方法执行完就结束了。而在小程序中，**页面是持久的**——`onLoad` 之后页面实例一直存在，直到 `onUnload`。这意味着你可以在 `data` 中维护页面级别的状态，而不需要像 Web 开发那样依赖 Session 或 Redis。

### 3.4 文件类型系统：四件套

每个小程序页面由四种文件组成，它们各司其职：

```
pages/index/
├── index.js      # 逻辑层 —— JavaScript
├── index.wxml    # 视图层 —— 模板
├── index.wxss    # 视图层 —— 样式
└── index.json    # 配置层 —— 页面配置
```

**各文件类型详解：**

#### WXML（WeiXin Markup Language）—— 模板文件

```xml
<!-- pages/index/index.wxml -->
<view class="container">
  <!-- 数据绑定 -->
  <text>{{message}}</text>

  <!-- 条件渲染（类比 Thymeleaf 的 th:if） -->
  <view wx:if="{{isLogged}}">
    <text>欢迎回来，{{userName}}</text>
  </view>
  <view wx:else>
    <text>请先登录</text>
  </view>

  <!-- 列表渲染（类比 Thymeleaf 的 th:each） -->
  <view wx:for="{{items}}" wx:key="id" class="item">
    <text>{{index}}. {{item.name}}</text>
    <text>价格：￥{{item.price}}</text>
  </view>

  <!-- 事件绑定 -->
  <button bindtap="onSubmit" type="primary">提交订单</button>

  <!-- 数据双向绑定（需要借助 model:） -->
  <input model:value="{{inputValue}}" placeholder="请输入" />
</view>
```

> **类比理解**
>
> | WXML 语法 | Thymeleaf 语法 | 说明 |
> | |---|---|---|
> | `{{变量名}}` | `${变量名}` | 数据输出 |
> | `wx:if="{{条件}}"` | `th:if="${条件}"` | 条件渲染 |
> | `wx:elif` / `wx:else` | `th:unless` | 条件分支 |
> | `wx:for="{{数组}}"` | `th:each="item : ${list}"` | 循环渲染 |
> | `wx:key="id"` | `th:key` | 列表渲染的 key 优化 |
> | `bindtap="方法名"` | `th:onclick` | 事件绑定 |

#### WXSS（WeiXin Style Sheets）—— 样式文件

```css
/* pages/index/index.wxss */

/* rpx 是小程序特有的响应式单位 */
/* 规定屏幕宽度为 750rpx，在 iPhone6 上 1rpx = 0.5px */
.container {
  padding: 30rpx;
  background-color: #f8f8f8;
}

.title {
  font-size: 36rpx;
  font-weight: bold;
  color: #333;
  margin-bottom: 20rpx;
}

/* 支持 @import 导入外部样式 */
@import "../../styles/common.wxss";

/* 支持 LESS/SASS 预处理器（需配置） */
```

> **Java 开发者注意点**
>
> `rpx`（responsive pixel）是小程序独有的响应式单位。它的设计理念是：**无论什么设备，屏幕宽度始终为 750rpx**。这样你只需要按照 750rpx 的宽度来设计布局，就能自动适配不同屏幕尺寸。
>
> 换算关系（以 iPhone6 为基准，屏幕宽度 375px）：
> - `1rpx = 0.5px`（iPhone6）
> - `1rpx = 0.554px`（iPhone6 Plus）
> - `1rpx = 0.421px`（iPhone5）
>
> 建议设计稿使用 750px 宽度，这样 1:1 对应 rpx 值即可。

#### JS —— 逻辑文件

```javascript
// pages/index/index.js
// 引入工具类（类比 Java 的 import）
const util = require('../../utils/util.js');

Page({
  // 页面数据（类比 Model/ViewModel）
  data: {
    message: 'Hello',
    items: [],
    inputValue: '',
    isLogged: false,
    userName: ''
  },

  // 生命周期函数
  onLoad(options) {
    // options.id 获取路由参数（类比 @PathVariable）
    const id = options.id;
    this.loadData(id);
  },

  // 事件处理函数
  onSubmit() {
    // 调用后端 API（类比 RestTemplate / Feign 调用）
    wx.request({
      url: getApp().globalData.baseUrl + '/api/order',
      method: 'POST',
      data: {
        inputValue: this.data.inputValue
      },
      header: {
        'content-type': 'application/json',
        'Authorization': 'Bearer ' + getApp().globalData.token
      },
      success(res) {
        if (res.statusCode === 200) {
          wx.showToast({ title: '提交成功', icon: 'success' });
        }
      },
      fail(err) {
        wx.showToast({ title: '网络异常', icon: 'none' });
      }
    });
  },

  // 私有方法（以下划线开头是约定俗成的命名规范）
  loadData(id) {
    wx.request({
      url: getApp().globalData.baseUrl + '/api/items/' + id,
      success: (res) => {
        this.setData({ items: res.data });
      }
    });
  }
})
```

#### JSON —— 页面配置文件

```json
{
  "navigationBarTitleText": "首页",
  "navigationBarBackgroundColor": "#1890ff",
  "navigationBarTextStyle": "white",
  "enablePullDownRefresh": true,
  "backgroundTextStyle": "dark",
  "usingComponents": {
    "custom-component": "/components/custom-component/index"
  }
}
```

> **类比理解**
>
> 页面级 JSON 配置类似于 Spring 中 Controller 级别的注解配置：
>
> | 小程序 JSON 配置 | Spring 注解/配置 | 说明 |
> | |---|---|---|
> | `navigationBarTitleText` | `@RequestMapping` 的元数据 | 页面标题 |
> | `enablePullDownRefresh` | 自定义 Filter/Interceptor | 启用下拉刷新 |
> | `usingComponents` | `@Autowired` 组件注入 | 引入自定义组件 |

### 3.5 四件套的 Java Web 类比总结

| 文件类型 | 角色 | Java Web 类比 | 核心职责 |
|---------|------|-------------|---------|
| **WXML** | 视图模板 | JSP / Thymeleaf 模板 | 定义页面结构和数据展示 |
| **WXSS** | 样式表 | CSS / Less / Sass | 控制页面外观和布局 |
| **JS** | 业务逻辑 | Spring Controller + Service | 处理业务逻辑、数据管理、API 调用 |
| **JSON** | 页面配置 | Spring 注解配置（`@Value`、`@ConfigurationProperties`） | 配置页面级别的行为和组件 |

> **Java 开发者注意点**
>
> 在 Java Web 开发中，你通常在一个 Controller 类中同时处理路由、业务逻辑和数据返回。而在小程序中，这些关注点被**文件级别**分离了：
>
> - 路由配置 → `app.json` 的 `pages` 数组
> - 页面结构 → `.wxml` 文件
> - 页面样式 → `.wxss` 文件
> - 业务逻辑 → `.js` 文件
> - 页面配置 → `.json` 文件
>
> 这种分离方式类似于前端框架（Vue 的单文件组件 `.vue`、React 的 JSX + CSS Modules）的设计理念。对于 Java 开发者来说，需要适应从"一个类搞定一切"到"多个文件协作"的思维转变。

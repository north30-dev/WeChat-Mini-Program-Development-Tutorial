# 微信小程序开发入门教程

> **适用读者**：具有 Java Web 开发经验（Spring Boot / Spring MVC）的后端开发者
> **学习目标**：从零掌握微信小程序开发，能够独立完成内容资讯类小程序项目
> **教程特色**：Java 视角对比、完整实战项目、企业级开发规范

## 第1章 微信小程序概述

### 1.1 小程序生态现状与发展趋势

微信小程序自 2017 年 1 月正式上线以来，已经发展成为一个庞大的应用生态。截至 2025 年，微信小程序的日活跃用户（DAU）已突破 **6 亿**，累计超过 **450 万**个小程序在平台上运行，覆盖了零售、餐饮、出行、政务、教育等几乎所有行业。

**小程序行业分布概况：**

| 行业类别 | 占比（约） | 典型场景 |
|---------|-----------|---------|
| 内容资讯 | ~18% | 新闻阅读、知识付费、自媒体 |
| 零售电商 | ~16% | 商城、拼团、直播带货 |
| 生活服务 | ~14% | 外卖、打车、酒店预订 |
| 工具类 | ~12% | 效率工具、记账、天气 |
| 政务民生 | ~10% | 社保查询、公积金、健康码 |
| 教育培训 | ~8% | 在线课程、题库、打卡 |
| 其他 | ~22% | 游戏、社交、医疗等 |

**发展趋势：**

1. **从"用完即走"到"用完再来"**：小程序已不再只是轻量工具，越来越多的企业将小程序作为核心业务载体，通过会员体系、消息推送等方式提升用户留存。
2. **商业化能力持续增强**：小程序直播、小程序广告、微信支付等能力让小程序具备了完整的商业闭环。
3. **跨端融合**：微信小程序与 PC 端、企业微信、微信搜一搜的联动越来越紧密，形成多端触达。
4. **技术能力开放**：越来越多的原生能力（蓝牙、NFC、生物识别等）向小程序开放，缩小了与原生 App 的能力差距。

### 1.2 小程序与原生 App、H5 的技术定位对比

| 维度 | 微信小程序 | 原生 App | H5 网页 |
|------|-----------|---------|---------|
| **开发语言** | WXML/WXSS/JS | Java/Kotlin (Android), Swift (iOS) | HTML/CSS/JS |
| **运行环境** | 微信客户端内 | 操作系统直接运行 | 浏览器 |
| **安装方式** | 无需安装，扫码即用 | 应用商店下载安装 | 浏览器访问 URL |
| **性能表现** | 接近原生（双线程渲染） | 最优 | 依赖浏览器内核 |
| **分发渠道** | 微信内搜索/分享/扫码 | 应用商店 | URL 链接 |
| **开发成本** | 中等（一套代码两端运行） | 高（需分别开发 Android/iOS） | 低 |
| **离线能力** | 有限（缓存机制） | 强 | 弱 |
| **更新方式** | 后台发布，用户无感更新 | 需用户手动更新 | 服务端更新即时生效 |
| **用户触达** | 微信生态内（模板消息、公众号关联） | 推送通知、应用商店 | 搜索引擎、链接分享 |

### 1.3 Java 开发者学习小程序的价值与职业路径

对于 Java Web 开发者而言，学习微信小程序开发具有以下价值：

1. **全栈能力补全**：Java 开发者通常擅长后端，掌握小程序前端开发后可以独立完成从后端 API 到前端页面的完整交付，成为真正的全栈开发者。
2. **职业竞争力提升**：市场上大量企业需要"Java 后端 + 小程序前端"的复合型人才，尤其在中小企业中，这种能力组合非常吃香。
3. **创业与副业机会**：小程序开发门槛相对较低，Java 开发者可以利用后端能力快速搭建完整的商业应用。
4. **技术思维拓展**：小程序的双线程架构、数据驱动视图等理念，有助于理解前端框架（如 Vue/React）的设计思想。

**典型职业路径：**

```
Java 后端开发 → Java + 小程序全栈开发 → 技术负责人/架构师
                                            ↓
                                    独立开发者/创业者
```

### 1.4 Java Web 应用 vs 微信小程序 对比表

| 维度 | Java Web 应用 | 微信小程序 |
|------|-------------|-----------|
| **运行环境** | JVM（Tomcat/Jetty 等容器） | 微信客户端（逻辑层 JSCore + 视图层 WebView） |
| **架构模式** | MVC（Controller → Service → DAO） | MVVM（数据驱动视图更新） |
| **部署方式** | 打包为 WAR/JAR，部署到服务器 | 上传代码包至微信后台，审核后发布 |
| **用户触达** | 用户主动访问 URL | 微信搜索、扫码、分享、公众号菜单、模板消息 |
| **后端交互** | 直接调用 Service 层 | 通过 HTTP/HTTPS 请求调用后端 API |
| **路由管理** | Spring MVC `@RequestMapping` | `app.json` 中的 `pages` 数组 + `wx.navigateTo` |
| **状态管理** | HttpSession / Redis / Spring Session | `getApp().globalData` / 页面 `data` |
| **数据绑定** | Thymeleaf / JSP 模板渲染 | WXML 数据绑定 `{{}}` 双花括号 |
| **样式方案** | CSS / Less / Sass | WXSS（支持 rpx 响应式单位） |
| **构建工具** | Maven / Gradle | 微信开发者工具（支持 npm） |

> **Java 开发者注意点**
>
> 小程序本质上是**前端应用**，它不运行在 JVM 上，而是运行在微信客户端的 JavaScript 引擎中。你熟悉的 Spring Boot 后端在小程序架构中扮演的是 **API 服务器**的角色。小程序通过 `wx.request()` 发起 HTTP 请求，与你用 Postman 测试 API 的方式类似。因此，学习小程序开发主要是学习一套**新的前端框架和开发范式**，而不是放弃 Java 技术栈。

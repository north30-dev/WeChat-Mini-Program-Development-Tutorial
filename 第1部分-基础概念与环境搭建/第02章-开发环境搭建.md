# 微信小程序开发入门教程

> **适用读者**：具有 Java Web 开发经验（Spring Boot / Spring MVC）的后端开发者
> **学习目标**：从零掌握微信小程序开发，能够独立完成内容资讯类小程序项目
> **教程特色**：Java 视角对比、完整实战项目、企业级开发规范

---

## 第2章 开发环境搭建

### 2.1 微信小程序账号注册与 AppID 获取

开发微信小程序的第一步是注册一个小程序账号并获取 AppID。

**注册流程：**

1. 访问 [微信公众平台](https://mp.weixin.qq.com/)，点击右上角"立即注册"。
2. 选择账号类型为"小程序"。
3. 填写邮箱、密码等基本信息完成注册。
4. 登录后台，在"开发" → "开发管理" → "开发设置"中找到 **AppID**。

> **类比理解**
>
> AppID 就像是你在微信生态中的"包名"。在 Java Web 开发中，你可能有 `com.example.myapp` 这样的包名来标识你的应用；在微信小程序中，AppID（形如 `wx1234567890abcdef`）就是微信用来唯一标识你小程序的"包名"。每个小程序必须绑定一个 AppID 才能使用微信的登录、支付等高级能力。

**账号类型说明：**

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| 个人账号 | 使用个人身份证注册 | 学习开发、个人工具类小程序 |
| 企业账号 | 需要企业营业执照 | 商业项目、需要微信支付等能力 |
| 测试号 | 开发者工具中可使用测试 AppID | 本地开发调试（部分功能受限） |

### 2.2 微信开发者工具安装与配置

微信开发者工具是官方提供的集成开发环境（IDE），集成了代码编辑、调试、预览、上传等全部功能。

**安装步骤：**

1. 访问 [微信开发者工具下载页面](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
2. 根据操作系统选择对应版本下载（支持 Windows 64 位、macOS）。
3. 安装完成后，使用微信扫码登录。

> **类比理解**
>
> 微信开发者工具 ≈ IntelliJ IDEA + Maven + Tomcat 的组合体。它既是代码编辑器，又是构建工具，还是运行容器。你不需要像 Java Web 开发那样分别安装 IDE、构建工具和应用服务器，微信开发者工具"一站式"搞定。

**推荐配置：**

- 将编辑器设置为 VS Code 模式（如果你更习惯 VS Code 的快捷键）
- 开启"热重载"（修改代码后自动刷新预览）
- 设置本地端口不校验（方便调用本地 Spring Boot 后端 API）

### 2.3 第一个小程序项目创建（Hello World）

1. 打开微信开发者工具，点击"新建项目"。
2. 填写项目信息：
   - **项目名称**：`hello-miniprogram`
   - **目录**：选择一个本地目录
   - **AppID**：填入你的 AppID（或选择"测试号"）
   - **开发模式**：小程序
   - **后端服务**：不使用云服务（我们使用自己的 Java 后端）
3. 点击"确定"，开发者工具会自动生成一个基础项目模板。

创建完成后，你会看到左侧的模拟器中已经显示了一个默认的欢迎页面。现在我们来修改它，实现一个 Hello World。

**修改 `pages/index/index.wxml`：**

```xml
<!-- pages/index/index.wxml -->
<view class="container">
  <text class="title">Hello World</text>
  <text class="subtitle">我的第一个微信小程序</text>

  <button bindtap="sayHello" type="primary">点击打招呼</button>

  <view wx:if="{{message}}" class="message">
    <text>{{message}}</text>
  </view>
</view>
```

**修改 `pages/index/index.js`：**

```javascript
// pages/index/index.js
Page({
  data: {
    message: ''
  },

  sayHello: function() {
    // 类比 Java 中的 this.message = "你好，小程序！"
    this.setData({
      message: '你好，小程序！当前时间：' + new Date().toLocaleString()
    });
  }
})
```

**修改 `pages/index/index.wxss`：**

```css
/* pages/index/index.wxss */
.container {
  display: flex;
  flex-direction: column;
  align-items: center;
  padding: 40rpx;
}

.title {
  font-size: 48rpx;
  font-weight: bold;
  color: #333;
  margin-bottom: 20rpx;
}

.subtitle {
  font-size: 28rpx;
  color: #888;
  margin-bottom: 60rpx;
}

.message {
  margin-top: 40rpx;
  padding: 30rpx;
  background-color: #f0f9eb;
  border-radius: 12rpx;
  color: #67c23a;
  font-size: 28rpx;
}
```

> **Java 开发者注意点**
>
> 注意 `this.setData()` 的用法。在 Java 中，你直接修改字段值即可；但在小程序中，**必须通过 `setData()` 来修改数据**，这样才能触发视图层的自动更新。这类似于 Vue.js 中的响应式数据机制。如果你直接 `this.data.message = 'xxx'`，页面是不会更新的。

### 2.4 项目目录结构解析

创建项目后，目录结构如下：

```
hello-miniprogram/
├── pages/                  # 页面目录（类比 Spring 的 Controller 层）
│   ├── index/              # 首页
│   │   ├── index.js        # 页面逻辑（类比 Controller + Service）
│   │   ├── index.wxml      # 页面模板（类比 JSP/Thymeleaf 模板）
│   │   ├── index.wxss      # 页面样式（类比 CSS）
│   │   └── index.json      # 页面配置（类比 Controller 级别的配置）
│   └── logs/               # 日志页面
│       ├── logs.js
│       ├── logs.wxml
│       ├── logs.wxss
│       └── logs.json
├── utils/                  # 工具类目录（类比 Java 的 util 包）
│   └── util.js
├── app.js                  # 小程序入口文件（类比 Spring Boot 的 Application 启动类）
├── app.json                # 全局配置文件（类比 application.yml）
├── app.wxss                # 全局样式文件（类比全局 CSS）
├── project.config.json     # 项目配置文件（类比 .idea 或 .classpath）
├── sitemap.json            # 站点地图配置（用于 SEO）
└── package.json            # npm 依赖配置（类比 pom.xml）
```

> **类比理解：项目结构对比**
>
> | 小程序目录/文件 | Java Web 对应物 | 说明 |
> | |---|---|---|
> | `pages/` | `src/main/java/.../controller/` | 页面是小程序的基本单元，每个页面一个目录 |
> | `utils/` | `src/main/java/.../util/` | 公共工具方法 |
> | `app.js` | `*Application.java` | 应用入口，管理全局生命周期 |
> | `app.json` | `application.yml` | 全局配置：路由、窗口、底部 TabBar 等 |
> | `app.wxss` | `src/main/resources/static/css/global.css` | 全局样式 |
> | `package.json` | `pom.xml` / `build.gradle` | 依赖管理（但小程序的依赖主要是前端库） |

### 2.5 app.json —— 小程序的 application.yml

`app.json` 是小程序的全局配置文件，它决定了小程序的整体行为。来看一个典型的配置：

```json
{
  "pages": [
    "pages/index/index",
    "pages/logs/logs",
    "pages/user/user",
    "pages/about/about"
  ],
  "window": {
    "navigationBarBackgroundColor": "#1890ff",
    "navigationBarTitleText": "我的小程序",
    "navigationBarTextStyle": "white",
    "backgroundColor": "#f5f5f5",
    "backgroundTextStyle": "dark"
  },
  "tabBar": {
    "color": "#999",
    "selectedColor": "#1890ff",
    "backgroundColor": "#fff",
    "list": [
      {
        "pagePath": "pages/index/index",
        "text": "首页",
        "iconPath": "images/home.png",
        "selectedIconPath": "images/home-active.png"
      },
      {
        "pagePath": "pages/user/user",
        "text": "我的",
        "iconPath": "images/user.png",
        "selectedIconPath": "images/user-active.png"
      }
    ]
  },
  "style": "v2",
  "sitemapLocation": "sitemap.json"
}
```

> **类比理解**
>
> | app.json 配置项 | application.yml 对应 | 说明 |
> | |---|---|---|
> | `pages` 数组 | `@RequestMapping` 路由注册 | 声明所有页面路径，第一个为首页 |
> | `window` | `spring.mvc.view.*` | 导航栏颜色、标题等窗口表现 |
> | `tabBar` | 前端路由配置 | 底部/顶部 Tab 导航栏 |
> | `style` | Spring Boot 版本号 | 指定组件样式版本 |
>
> 注意一个关键区别：`pages` 数组中的**第一个路径就是小程序的启动首页**。这类似于 Spring Boot 中 `@SpringBootApplication` 所在包的扫描起点。

### 2.6 依赖管理：npm 在小程序中的使用

小程序没有 `pom.xml`，但支持通过 npm 管理前端依赖。

**初始化 npm：**

```bash
# 在项目根目录执行
npm init -y
```

**安装第三方库（示例：日期处理库 dayjs）：**

```bash
npm install dayjs --save
```

**在 `app.json` 中构建 npm：**

打开微信开发者工具 → "工具" → "构建 npm"，工具会自动将 `node_modules` 中的包打包到 `miniprogram_npm` 目录中。

**在页面中使用：**

```javascript
// pages/index/index.js
const dayjs = require('dayjs');

Page({
  onLoad() {
    const now = dayjs().format('YYYY-MM-DD HH:mm:ss');
    this.setData({ currentTime: now });
  }
})
```

> **Java 开发者注意点**
>
> | 概念 | Maven/Gradle | 小程序 npm |
> | |---|---|---|
> | 配置文件 | `pom.xml` / `build.gradle` | `package.json` |
> | 依赖仓库 | Maven Central | npm Registry |
> | 依赖目录 | 本地 `.m2` 仓库 | `node_modules` |
> | 构建产物 | WAR/JAR | `miniprogram_npm/` |
> | 依赖范围 | compile/runtime/test | 无 scope 概念，全部打包 |
>
> 与 Maven 不同，npm 没有类似 `provided` 或 `test` 的 scope 概念，所有安装的依赖都会被打包进小程序代码包中。因此要注意控制依赖体积，小程序的代码包总大小限制为 **2MB**（使用分包可扩展至 20MB）。
